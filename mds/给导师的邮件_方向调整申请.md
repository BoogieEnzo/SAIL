**邮件主题**：关于实习方向的调整申请

**收件人**：[导师邮箱]

**正文**：

老师您好，

感谢您为我提供这次实习机会。经过这段时间的学习和思考，我想向您汇报一下我对当前实习方向（SAIL/ACT）的理解，并希望能与您讨论一下方向调整的可能性。

**对当前方向的理解**

通过学习和调研，我了解到 SAIL/ACT 方向主要是：
- SAIL：用形式化语言定义 RISC-V 指令集的行为规范，作为硬件实现的参考标准
- ACT：编写测试用例验证硬件实现是否符合 RISC-V 标准

这是一个**硬件验证**方向，工作重点在于确保硬件实现的正确性和符合性。

**我的背景和兴趣**

回顾我的学习和项目经历，我发现自己更感兴趣和更擅长的是**系统软件和平台化基础设施**方向：

1. **操作系统内核开发**：我完成了 rCore（Rust）和 Nanos-lite（C）两个完整的内核实现，包括虚拟内存、进程管理、文件系统、设备驱动等核心模块。这些经历让我对操作系统底层机制有了深入理解。

2. **模拟器开发**：我参与开发了 NEMU RISC-V 指令集模拟器，实现了取指-译码-执行全流程，并配合 DiffTest 进行指令级状态比对。这让我对模拟器的工作原理有了实际经验。

3. **硬件适配和驱动开发**：我在项目中实现了 UART、块设备、键盘、鼠标、GPU 等外设驱动，并配合 QEMU 模拟设备进行开发调试。这让我对硬件接口和驱动开发有了实践经验。

基于这些经历，我发现自己更倾向于**让软件适配硬件**的工作，而不是**验证硬件是否符合标准**的工作。

**希望的方向**

如果可能的话，我希望能在以下方向继续学习和贡献：

1. **操作系统移植和适配**：Linux Kernel RISC-V 移植、操作系统发行版（如 openEuler、Deepin）的 RISC-V 适配工作

2. **QEMU 模拟器开发**：基于我的 NEMU 开发经验，希望能参与 QEMU RISC-V 相关的开发工作

3. **Hypervisor 开发**：Bao-Hypervisor、arceos-hypervisor 等虚拟化平台的 RISC-V 支持

4. **平台化软件基础设施**：BSP 开发、引导程序（U-boot、OpenSBI）适配、嵌入式系统开发等

这些方向更符合我的技术背景和职业规划，也能让我更好地发挥已有的经验和能力。

**我的想法**

我理解 SAIL/ACT 方向对 RISC-V 生态建设的重要性，也感谢您为我提供这个机会。但考虑到我的技术背景和兴趣方向，我希望能与您讨论一下是否有调整到更匹配方向的可能性。
