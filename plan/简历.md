## 2. 实习经历

### **2.1 RISC-V 操作系统内核开发 | 清华大学陈渝教授课题组（寒假实习，线上）**

**项目描述：**  
基于 rCore 教程在 RISC-V 上实现操作系统内核，覆盖批处理、多任务调度、虚拟内存、进程管理、文件系统、同步互斥、设备驱动等核心模块。Linux 开发环境下使用 QEMU RISC-V virt 仿真平台运行与调试内核，Rust 进行内核模块设计与实现。

**主要工作：**
- **开发与调试：** Linux 环境下使用 RISC-V 交叉编译工具链，QEMU 仿真运行内核，GDB 远程调试
- **批处理系统：** 实现应用加载与执行，ELF 文件解析与内存布局，支持多应用批处理运行
- **多任务调度：** 实现协作式与抢占式多任务，基于时钟中断的调度，任务上下文切换与内核栈管理，理解调度对系统响应与资源利用的影响
- **虚拟内存管理：** RISC-V Sv39 三级页表实现，内核与用户地址空间隔离，页表映射与页错误处理，具备内存管理与地址空间设计基础
- **进程管理：** 进程控制块（PCB）与任务控制块（TCB），fork/exec/wait/exit 系统调用，进程间通信（IPC）机制
- **文件系统：** 基于块设备的简单文件系统，块缓存机制，文件读写与目录管理，支持用户态 shell 交互
- **同步互斥：** 自旋锁与阻塞锁实现，信号量机制，解决多线程竞争条件
- **设备驱动：** UART、块设备、键盘、鼠标、GPU 等外设驱动，基于中断的设备 I/O 处理；配合 QEMU 模拟设备进行开发与联调

### **2.2 RISC-V 软硬件栈：模拟器、抽象层与内核的协同设计 | 中国开源芯片研究院（2025.08 - 2025.10，深圳）**

**项目描述：**  
参与 RISC-V 软件栈开发：**NEMU** 指令集模拟器 → **AM** 抽象机（TRM + IOE）→ **Nanos-lite** 最小内核 → **Navy** 用户态运行时。C 环境下完成 I/O 与设备抽象（端口 I/O、MMIO）、批处理与分时多任务，支持 NTerm、仙剑奇侠传等在模拟器上运行。

**主要工作：**
- **NEMU 模拟器：** 实现 RV32IM 取指-译码-执行全流程；端口 I/O 与 MMIO 两种编址方式，仿真串口、时钟、键盘、VGA 等设备；理解状态机视角下 I/O 与设备访问
- **AM 与 IOE：** 在 AM 中实现 IOE，将设备访问抽象为架构无关 API（时钟、键盘、GPU 等），支撑 NEMU 与 native 多平台；配合 **DiffTest** 与 Spike 做指令级状态比对，区分硬件/软件 bug
- **工程化与编译链接：** 链接脚本控制段布局与强/弱符号，解决 Klib 与 glibc 冲突；掌握从 C 到可执行文件的完整编译链接与 ELF 加载流程
- **Nanos-lite 与虚拟内存：** RISC-V Sv32 两级页表；filesz/memsz 与 BSS 清零、段重叠页复用；批处理加载用户程序与系统调用接口
- **分时多任务与栈切换：** 时钟中断驱动的抢占式调度；内核栈与用户栈切换、CTE 中 pp/ksp/np/usp 维护，支持多用户进程分时运行；实现 execve 与 mainargs 传递，Navy 下 Shell/NTerm 与应用程序协同

## 3. 项目经历

### **3.1 AI 应用服务平台 (CppAIService)**

**项目描述：**  
基于 C++ 自研 HTTP 服务框架，实现支持多模型对话、图像识别与语音交互的 AI 应用平台。依托 Muduo 网络库与 RabbitMQ 消息队列实现高并发异步推理，集成 MySQL 持久化、ONNX Runtime 本地推理与百度语音 API，使用 CMake 构建并支持 Docker 容器化部署。

**主要工作：**
- **AI 对话系统：** 在自研 HTTP 框架中集成多模型对话接口（阿里百炼、豆包、RAG、MCP 等），采用策略模式与工厂模式设计 AIStrategy 与 StrategyFactory，支持多轮对话与会话记忆，实现模型动态切换与扩展
- **图像识别服务：** 基于 ONNX Runtime + OpenCV 部署轻量级分类模型，实现端到端识别流程（图像加载 → 预处理 → 推理 → 标签解析），支持文件路径与内存 Buffer 两种输入方式
- **高并发与异步处理：** 基于 Muduo 网络库处理 HTTP 请求，将 AI 聊天与图像识别任务通过 RabbitMQ 队列分发，配合 RabbitMQThreadPool 异步执行模型推理与数据库写入，避免主线程阻塞
- **用户与会话管理：** 实现登录、注册、退出及历史记录同步，基于 Session 机制完成用户状态校验；AIUploadHandler 等模块实现基于 Session 的权限校验与 JSON 错误响应
- **MCP 风格工具调用：** 参考 Model Context Protocol 思想，实现 AIToolRegistry 工具注册与调用机制，支持 getWeather、getTime 等工具，为语音识别等扩展预留接口
- **语音识别与合成 (ASR+TTS)：** 集成百度语音 API，实现 AISpeechProcessor 的语音识别与合成，支持 PCM/MP3 格式与异步轮询
- **数据持久化与部署：** 使用 MySQL Connector/C++ 实现用户与对话历史持久化，Docker 整合 MySQL、RabbitMQ 等组件，支持一键部署

**主要工作：**
- **可靠字节流：** ByteStream、32 位序列号包装与接收端重组器，处理乱序/重叠/重复
- **链路层：** 以太网帧封装/解析，ARP 请求/应答与缓存老化
- **网络层：** IPv4 封装/解析，最长前缀匹配与转发
- **TCP：** 三次握手与四次挥手、序列号可靠传输与 RTO 重传、接收窗口与流量控制、零窗口探测
