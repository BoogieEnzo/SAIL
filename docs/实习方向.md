## 考核要求

- 提交 PR
  - 必须通过 CI, 除非由 lean/rocq 导致
  - 保证一定的代码质量
  - 不能太简单，例如修复 typo 等
  - 注意确保不要和当仓库其他 PR 功能撞车
  - 如果 PR 质量不合格，则视为考核失败
- PR 提交后，还需要一个做一个技术报告
  - 时间固定每周三下午 3 点，腾讯会议，准备 PPT, 时长 30 分钟左右
  - 可以讲基础知识，问题背景，思维过程，难点，解决方法

## ✅ 最终选择：为Zabha添加ACT测试支持

**扩展名称**：Zabha - Byte and Halfword Atomic Memory Operations

**任务类型**：为ACT（riscv-arch-test）开发测试支持

**状态**：
- Zabha已批准（2024年4月）
- sail-riscv已实现框架
- **ACT缺少Zabha测试覆盖** ✅ 这是我们的机会

**规范链接**：https://docs.riscv.org/reference/isa/unpriv/zabha.html

**复杂度评估**：
- ✅ **中等难度**：为16条原子操作指令编写测试用例
- ✅ **指令明确**：AMOADD/AND/OR/XOR/SWAP/MIN/MAX等字节和半字版本
- ✅ **你的优势**：有A扩展经验，熟悉原子操作和测试

**工作内容**：
- 在ACT仓库中为Zabha扩展创建测试用例
- 覆盖所有16条指令（8种操作 × 字节/半字）
- 验证对齐、非对齐、边界情况等

**参考实现**：
- `riscv-arch-test/riscv-test-suite/` - 现有测试结构
- `riscv-arch-test/riscv-test-suite/rv64i_m/A/` - A扩展示例
- Zabha规范文档中的指令定义

**RISC-V规范**：
- GitHub: https://github.com/riscv/riscv-zabha
- 文档: https://docs.riscv.org/reference/isa/unpriv/zabha.html

**选择理由**：
- ✅ 规范完整（有独立文档）
- ✅ 已批准（2024年4月）
- ✅ 无冲突（ACT缺少Zabha测试）
- ✅ 复杂度适中（1-2周可完成）
- ✅ 风险低（不修改核心代码，只添加测试）
- ✅ 汇报友好（测试逻辑容易讲清楚）
