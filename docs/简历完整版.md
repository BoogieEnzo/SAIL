**项目描述：** 基于 rCore 教程在 RISC-V 上实现操作系统内核，覆盖批处理、多任务调度、虚拟内存、进程管理、文件系统、同步互斥、设备驱动等核心模块；在 Linux 环境使用 QEMU RISC-V virt 运行与调试，Rust 进行内核模块设计与实现。

**主要工作：**
- **开发与调试：** Linux 环境下使用 RISC-V 交叉编译工具链，QEMU 仿真运行内核，GDB 远程调试
- **批处理系统：** 实现应用加载与执行，ELF 文件解析与内存布局，支持多应用批处理运行
- **多任务调度：** 实现协作式与抢占式多任务，基于时钟中断的调度，任务上下文切换与内核栈管理，理解调度对系统响应与资源利用的影响
- **虚拟内存管理：** RISC-V Sv39 三级页表实现，内核与用户地址空间隔离，页表映射与页错误处理，具备内存管理与地址空间设计基础
- **进程管理：** 进程控制块（PCB）与任务控制块（TCB），fork/exec/wait/exit 系统调用，进程间通信（IPC）机制
- **文件系统：** 基于块设备的简单文件系统，块缓存机制，文件读写与目录管理，支持用户态 shell 交互
- **同步互斥：** 自旋锁与阻塞锁实现，信号量机制，解决多线程竞争条件
- **设备驱动：** UART、块设备、键盘、鼠标、GPU 等外设驱动，基于中断的设备 I/O 处理；配合 QEMU 模拟设备进行开发与联调

### **3.4 AIOS（Agent 操作系统）前沿架构学习与落地实践**

**项目描述：** **AIOS**（AI Agent Operating System）代表「LLM as OS、Agents as Apps」的前沿方向：Kernel 作为 OS 之上的抽象层，统一管理 Agent 所需的 LLM、调度、上下文切换、内存、存储与工具；SDK（Cerebrum）供开发者构建与运行 Agent，Agent 经 SDK 向 Kernel 发起请求，Kernel 将请求转为 syscall 链并调度到对应模块；在系统学习该架构与设计思想的基础上，完成本地部署与扩展实践，形成对 Agent 操作系统范式的理解。

**主要工作：**
- **架构理解与梳理：** 厘清 Kernel（LLM Core、Context Manager、Memory Manager、Tool Manager 等）与 SDK 的边界，理解 Agent 请求经 syscall 链调度、分派至各模块的流程；区分 Local Kernel 与 Remote Kernel 等部署模式及适用场景
- **部署与配置：** 本地搭建 Kernel + Cerebrum，配置多后端 LLM（Ollama/vLLM），跑通 Terminal UI 与示例 Agent，整理可复现的安装与启动文档
- **扩展实践：** 基于 syscall/tool 接口接入自定义工具并在 Kernel 上验证调用链，巩固对「Agent — SDK — Kernel — 模块」全链路的理解

**项目描述：** 研读论文与开源实现 **LLMCompiler**（An LLM Compiler for Parallel Function Calling），理解其将复杂问题分解为可并行/可依赖的任务图，并以 Planner→Task Fetching→Executor 的流水线编排多工具调用，实现降低延迟与成本、提升可用性。

**主要工作：**
